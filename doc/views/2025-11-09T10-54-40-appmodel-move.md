# AppModel moved to application layer

Date: 2025-11-09T10-54-40

Summary:
- Moved `AppModel` from `src/data/mod.rs` to `src/application/app_model.rs`.
- Rationale: keep the data module free from application/rendering coupling and align with lean application architecture.
- Transitional re-export kept in `data::mod` to minimize disruption: `pub use crate::application::app_model::AppModel;`.

Impact:
- No functional changes. Existing imports using `crate::data::AppModel` continue to work.
- Builds for native and WASM remain unaffected.

Usage example (reference only):

```rust
// no_run
use kepler_wgpu::application::AppModel;
use kepler_wgpu::data::ct_volume::CTVolume;

fn initialize_model(vol: CTVolume) -> AppModel {
    // Function-level comment: Demonstrates creating and populating AppModel.
    let mut model = AppModel::new();
    let _ = model.load_volume(vol);
    model
}
```

Notes:
- Logging follows workspace rules (INFO default; DEBUG for development).
- No UI changes involved.
